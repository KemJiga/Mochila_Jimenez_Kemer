¿Por qué un lenguaje de programación sólo puede utilizarse en algunos
sistemas operativos y en otros no?.
R/ Un lenguaje de programación puede estar diseñado para funcionar en sistemas operativos específicos debido a varias razones:
a) Dependencia de bibliotecas y APIs: Los sistemas operativos tienen diferentes bibliotecas y conjuntos de interfaces de programación de aplicaciones (APIs). Si un lenguaje de programación utiliza funciones o características específicas de una biblioteca o API que no está disponible en otro sistema operativo, no se podrá ejecutar en ese sistema operativo.

b) Compilación y ejecución: Algunos lenguajes de programación se compilan en código de máquina específico de un sistema operativo antes de ejecutarse. Esto significa que el código fuente debe ser procesado y traducido al código de máquina del sistema operativo en el que se ejecutará. Si no hay un compilador disponible para un lenguaje en un sistema operativo en particular, no se podrá ejecutar en ese sistema operativo.

c) Restricciones de licencia: Algunos lenguajes de programación pueden tener restricciones de licencia que limitan su uso a ciertos sistemas operativos. Por ejemplo, puede haber licencias comerciales que requieran el pago de regalías para utilizar el lenguaje en ciertos sistemas operativos.

¿Qué tipo de máquina virtual es virtualBox?.
R/ VirtualBox crea una "máquina virtual de sistema" que simula un entorno de hardware completo, lo que permite ejecutar otros sistemas operativos en una ventana separada o en pantalla completa dentro del sistema operativo principal.

Si tengo más de una máquina virtual instalada, y una se rompe, ¿esto afecta a las
demás? ¿por qué?
R/ Si una máquina virtual se rompe, generalmente no afecta a las demás máquinas virtuales. Cada máquina virtual se ejecuta de forma aislada del sistema operativo anfitrión y de otras máquinas virtuales.
